Frequently Asked Questions

The difference between var,let,and const
In JavaScript, var is function-scoped, hoisted with undefined, and allows both redeclaration and reassignment, making it less reliable. let is block-scoped, hoisted but not initialized, and allows reassignment but not redeclaration, making it a safer choice for mutable variables. const is also block-scoped, hoisted but not initialized, and cannot be reassigned or redeclared, making it ideal for constants. Since var can lead to unintended bugs, it is generally avoided. The best practice is to use const by default and let only when a variable needs to be reassigned
The difference map(),forEach(),and filter()
In JavaScript, map(), forEach(), and filter() are used for iterating over arrays but serve different purposes. map() returns a new array by transforming each element, making it useful for data modification. forEach() simply executes a function for each element without returning a new array, mainly for side effects like logging. filter() returns a new array containing only elements that satisfy a specific condition, helping to extract relevant data. Use map() for transformations, forEach() for side effects, and filter() for selective filtering
Explain arrow functions and how they are different from regular functions
Arrow functions in JavaScript are a shorter syntax for writing functions using =>. Unlike regular functions, they do not have their own this value; instead, they inherit this from their surrounding scope, making them useful for maintaining context in callbacks and event handlers. They also do not have their own arguments object and cannot be used as constructors. Arrow functions are ideal for concise one-liners and functional programming but should be avoided when a function requires this binding or arguments. Regular functions, on the other hand, provide more flexibility for dynamic this and can be used in all contexts.
How JavaScript Promises work
JavaScript Promises handle asynchronous operations by representing a value that may be available now, later, or never. A Promise has three states: pending, fulfilled, or rejected. When resolved, .then() executes the success handler, and if rejected, .catch() handles errors. Promises help avoid callback hell, making async code cleaner and more manageable. They are commonly used for API calls, file handling, and timers. The async/await syntax simplifies working with Promises, making code even more readable.
How closures work in JavaScript
Closures in JavaScript occur when a function retains access to its parent scope, even after the parent function has finished executing. This happens because inner functions remember the variables of their outer functions due to lexical scoping. Closures are useful for data encapsulation, creating private variables, and callback functions. They help in maintaining state between function calls without polluting the global scope. A common use case is in event handlers, timers, and function factories.
